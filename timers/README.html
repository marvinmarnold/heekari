Description

This example application demonstrates timers and timer routines using CSR100x and CSR101x development boards.

This ReadMe is not intended to describe procedures that are covered in the xIDE user guide such as building projects in xIDE etc.

For further information, please read the SDK support documentation.

Prerequisites

Hardware requirements

This application requires:

  • A CSR µEnergy CSR1000™/CSR1010™ development board (CNS10004), or
    a CSR µEnergy CSR1001™ development board (CNS10017), or
    a CSR µEnergy CSR1011™ development board (CNS12016)
    (see Configuration Details)
  • A CSR µEnergy USB to SPI adapter board (CNS10020) plus mini-USB cable
  • A CSR µEnergy 8-way adapter cable
  • A host PC system with a USB connection

To download the application to the development board, connect the USB cable to the PC, and the USB to SPI adapter to the development board using the 8-way adapter cable.

Software requirements

This application requires a terminal client e.g. HyperTerminal, PuTTY, etc.

Running the application

  1. Open the timers application workspace (timers.xiw) in xIDE.
  2. Select a build configuration (Debug or Release) from the drop-down field in the toolbar.
  3. Specify which transport the device is connected to by opening the Debug menu and clicking on Transport...
  4. Build (F7) the application.
  5. Run (F5) the application.

Controlling the Application

Once executed the application runs until stopped, reset or powered down.
This application demonstrates the functioning of timers by displaying messages over the UART after 1 second, then after 2 seconds, then after 1 second and so on, by using a single timer.

Configuration Details

Configure your terminal client to connect to the virtual UART exposed by the USB to SPI adapter board (CNS10020). Default terminal client settings are 2400-8-N-1.

NOTE: Using this example on the CSR1000/CSR1010 development board requires the UART to be enabled as described in the quick start guide.

Configuration Store Settings

Default keys can be used.
